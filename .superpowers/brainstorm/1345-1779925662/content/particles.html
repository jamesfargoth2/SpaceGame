Particle Effects Upgrade

The existing ParticleRenderSystem already handles billboard particles via DecalBatch — the upgrade adds a real texture atlas, GPU depth-sorting, and a second emitter type for mesh-based particles.

◎ BillboardEmitter — soft, glowing, atmospheric
engine_exhaust — additive blend

◈ MeshParticleEmitter — sharp, physical, reactive

engine_exhaust.json (Billboard)

"type": "BILLBOARD",
"sprite": "flame",
"blendMode": "ADDITIVE",
"maxParticles": 200,

"lifetime": { "min": 0.3, "max": 0.8 },
"size": { "start": 0.4, "end": 0.0, "curve": "EASE_OUT" },
"color": { "start": "#ff8800ff", "end": "#ff220000" }

impact_sparks.json (Mesh)

"type": "MESH",
"mesh": "spark_line",
"maxParticles": 50,
"emitOnce": true,
"speed": { "min": 3.0, "max": 12.0 },
"lifetime": { "min": 0.2, "max": 0.6 },
"gravity": -9.8,
"bounce": 0.3
Depth sorting: Each frame, all billboard particles across all active emitters are collected into a single array and sorted back-to-front by camera distance — one Arrays.sort() call, then one DecalBatch.flush(). Mesh particles render separately via ModelBatch with depth write enabled — no sort needed. Existing ParticleRenderSystem and all current VFX JSON files stay compatible; only the texture paths get populated.
