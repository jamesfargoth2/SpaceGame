Additive layers are applied on top of the base pose. Each is independently enabled/disabled via events.
Float Parameters (driven by ECS components each frame)
moveSpeed 0→1 strafeX −1→1 verticalVelocity aimPitch −1→1 swimSpeed 0→1 airborne bool crouching bool swimming bool
Each frame
1. ECS system writes float params from physics components
2. BlendTree.evaluate() traverses nodes, computes weights
3. Each ClipNode samples its animation at its local time
4. Node transforms are blended (lerp position, slerp rotation)
5. Additive deltas applied on top
6. Final pose written to ModelInstance skeleton
Key difference from libGDX AnimationController
libGDX's built-in controller only plays one clip at a time (with crossfade). The blend tree manually samples multiple clips in parallel and mixes their node transforms directly — gives us full locomotion blending and additive layers simultaneously.
