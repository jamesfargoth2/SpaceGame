Hysteresis: Tier transitions use separate upgrade/downgrade distances (e.g., upgrade to T0 at 12m, downgrade back to T1 at 18m) to prevent visual popping when the player hovers near a threshold. All thresholds are JSON-configurable per category.

Rendering Strategy Per Asset Type

Foliage — GPU Instancing

FoliageRenderSysteminstancedwind shader

World divided into tiles. Each tile holds an instance buffer (Matrix4 array) for every foliage type in it. One glDrawArraysInstanced call per foliage type per tile. Wind: vertex shader applies a sine-wave XZ displacement scaled by vertex Y and a time uniform — grass bends at the tips, trunks barely move. Tile culling: skip any tile whose AABB is outside the camera frustum. No per-blade physics — pure visual.

Props & Interior Objects — Pooled Instances + Selective Instancing

PropCatalogPool<ModelInstance>PropPlacementSystem

PropCatalog JSON registry: each prop has an ID, mesh path, LOD tier paths, and category tags (SEATING, CONSOLE, DECORATION, STORAGE, TECH). Pooled ModelInstances via libGDX Pool<T> — reused when props enter/leave visibility. For high-repetition props (chairs, crates, lockers): GPU instancing. For hero props (captain's chair, unique consoles): individual ModelInstance. PropPlacementSystem populates rooms at interior-enter time based on room archetype + tag filter.

Buildings — Modular Assembly + Static Bake

BuildingAssemblersnap pointsstatic mesh

Each building panel (wall section, floor, roof, doorframe, window) is a glTF model with defined snap point metadata (attachment position + orientation). BuildingAssembler reads a building definition JSON, instantiates panels, connects them via snap points, then merges into a single static mesh (one ModelInstance per building, one draw call). Once assembled, the building never re-draws individual panels. Procedurally varied via panel swapping per seed — different wall styles, window counts, roof shapes.
