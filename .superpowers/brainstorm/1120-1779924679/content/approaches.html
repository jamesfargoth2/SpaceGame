Atmospheric Rendering: 3 Approaches

Full Rayleigh+Mie scattering, per-planet variation, day/night cycle

A

Single-Pass Analytic Scattering (Recommended)

Compute Rayleigh + Mie scattering analytically in a single fullscreen-quad fragment shader. Uses the Nishita/Preetham model with lookup approximations instead of precomputed LUTs.

Pros

  • Single shader pass — no precomputation or texture management
  • Per-planet parameters feed directly into uniforms (scattering coefficients from atmosphere composition)
  • Day/night is just changing the sun direction uniform
  • Volumetric cloud layer integrates naturally into the same ray-march
  • Fog is derived from the same scattering model — no horizon bleed

Cons

  • Moderate GPU cost per fragment (optical depth integral)
  • Analytic approximations slightly less accurate than precomputed LUT at extreme angles
B

Precomputed LUT Scattering (Bruneton)

Precompute transmittance and in-scatter into 2D/3D textures at planet load time. Runtime shader samples these LUTs for fast, accurate sky rendering.

Pros

  • Most physically accurate — handles multiple scattering
  • Very fast at runtime (just texture lookups)
  • Industry standard (used by Unreal, many AAA titles)

Cons

  • Requires precomputation step per planet (compute shaders or CPU bake)
  • More complex texture management — 3D textures for in-scatter
  • Harder to integrate volumetric clouds (separate pass needed)
  • OpenGL 4.x compute shaders needed for GPU precomputation
C

Hybrid: Analytic Sky + Raymarched Clouds

Analytic Rayleigh/Mie for the sky dome (no precomputation), then a separate low-resolution raymarched cloud pass composited on top. Two passes but each is focused and optimized.

Pros

  • Each pass is simpler and more debuggable
  • Cloud pass can be rendered at half/quarter resolution for performance
  • Clouds can be temporally reprojected for smooth animation
  • Easy to disable clouds on low-spec hardware

Cons

  • Two shader passes instead of one
  • Cloud compositing needs depth-aware blending
  • Slightly more code to manage
