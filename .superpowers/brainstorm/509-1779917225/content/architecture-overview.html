Swimming & Water Systems — Architecture Overview

How the new systems layer onto the existing codebase

System Architecture Diagram

Key Integration Points

  • WaveSystem — already exists, extended by WeatherSystem to drive amplitude/chop dynamically
  • SwimmingSystem — takes over from PlayerMovementSystem when player enters water
  • UnderwaterSystem — manages depth zones, visibility, and pressure as player descends
  • Components — all new ECS components, data-driven from JSON config files
  • Events — cross-system communication via existing EventBus (no direct references)
