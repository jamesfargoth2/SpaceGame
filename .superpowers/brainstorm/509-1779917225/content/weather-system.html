How Weather Drives Waves

The WeatherSystem doesn't create waves directly — it modifies the existing WaterBodyComponent.waves array by interpolating wave parameters toward a target profile. Each weather state defines a wave profile in weather_profiles.json:

  • Wave count — calm uses 2 components (gentle swell); storm uses 5–6 (swell + chop + cross-waves)
  • Amplitude envelope — lerped smoothly over the phase transition duration
  • Steepness (Gerstner Q) — calm: 0.1 (sine-like); storm: 0.6–0.8 (peaked, near-breaking)
  • Direction spread — calm: waves mostly aligned; storm: multi-directional for chaotic surface
  • Current velocity — wind drives WaterBodyComponent.currentVelocity, affecting swim drag and ship drift

Storm Cells — Spatial Model

Storms are localized regions, not global state. Each storm is an entity with a StormCellComponent containing a center position (galaxy-space), radius (1–5 km), and its own weather state. Wave parameters blend based on distance to the nearest storm cell:

  • Inside radius — full storm wave profile
  • Edge band (outer 30%) — lerp between storm and ambient profile. This creates the "wall of rain" sailing-into-a-storm feel.
  • Outside — ambient calm profile
  • Movement — storm cells drift with wind direction at 5–20 m/s. A storm can chase you.

Impact on Swimming & Ships

Weather Swimming Effect Ship Effect
Calm Gentle bobbing, easy surface swimming Smooth sailing, minimal roll
Building Stronger bobbing, current starts pushing Noticeable sway, waves lap deck
Storm Waves can submerge swimmer, strong currents, stamina drains 2x, visibility drops at surface Heavy roll/pitch, waves crash over deck, water ingress, risk of capsize
Subsiding Large swells but less chop, currents fading Long rolling waves, gradually calming
